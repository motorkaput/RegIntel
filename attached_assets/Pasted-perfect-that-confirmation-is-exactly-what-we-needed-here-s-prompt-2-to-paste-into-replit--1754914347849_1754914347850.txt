2. **SQL for RLS** in a new file `prisma/rls.sql`:

   * `ALTER TABLE … ENABLE ROW LEVEL SECURITY;` for all tenant-scoped tables and `tenants`.
   * A read policy on `tenants` allowing `SELECT` only the row whose `id = current_setting('app.current_tenant_id')::uuid`.
   * A default tenant isolation policy for each tenant-scoped table:

     ```
     CREATE POLICY tenant_isolation_select ON <table> FOR SELECT
       USING (tenant_id = current_setting('app.current_tenant_id')::uuid);
     CREATE POLICY tenant_isolation_mod ON <table> FOR INSERT WITH CHECK
       (tenant_id = current_setting('app.current_tenant_id')::uuid);
     CREATE POLICY tenant_isolation_update ON <table> FOR UPDATE
       USING (tenant_id = current_setting('app.current_tenant_id')::uuid)
       WITH CHECK (tenant_id = current_setting('app.current_tenant_id')::uuid);
     CREATE POLICY tenant_isolation_delete ON <table> FOR DELETE
       USING (tenant_id = current_setting('app.current_tenant_id')::uuid);
     ```
   * **Role-aware write limits**:

     * `users`: only `admin` can insert/update/delete.
     * `goals`: `admin, org_leader, functional_leader` can insert/update; others read.
     * `projects`: `admin, org_leader, functional_leader, project_lead` can insert/update; team\_member read.
     * `tasks`: `admin, org_leader, functional_leader, project_lead` can insert/update; `team_member` can update only assigned tasks’ status/self-score fields.
   * Implement role checks via a security helper view or function using the JWT claims propagated by the app (e.g. `current_setting('app.current_role', true)`), and reference it in policies, e.g.:

     ```
     USING (tenant_id = current_setting('app.current_tenant_id')::uuid
        AND current_setting('app.current_role') IN ('admin','org_leader','functional_leader'))
     ```
   * Add a note at the top explaining the app middleware must run:

     ```
     SELECT set_config('app.current_tenant_id', <tenant_uuid>, true);
     SELECT set_config('app.current_role', <role>, true);
     SELECT set_config('app.current_user_id', <user_uuid>, true);
     ```
